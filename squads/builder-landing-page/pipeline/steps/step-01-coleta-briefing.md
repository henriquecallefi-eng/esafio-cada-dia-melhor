# Step 01 — Coleta de Briefing

**Type:** Checkpoint  
**Agent:** User  
**Output:** Briefing Data (entrada para Beatriz)

## Descrição

Este é o checkpoint inicial. O usuário fornece informações básicas sobre o produto/serviço que será usado para criar a landing page. Essas informações alimentam todos os agentes subsequentes.

## Input Requerido

### Informação Básica

1. **Nome do Produto/Serviço**  
   Ex: "HUB R.E.C. — Acelerador de Negócios"

2. **Descrição Curta** (1-2 parágrafos)  
   O que é, para quem, por quê?

3. **Diferencial Principal**  
   O que torna único? (ex: "Mentoria 1-on-1 + consultoria de marketing")

4. **Público-alvo** (pode ser vago, Beatriz refina)  
   Ex: "Empreendedores com negócio em crescimento"

5. **Objetivo da Landing Page**  
   - Conversão (email, agendamento, compra)?
   - Educação (webinar, whitepaper)?
   - Waitlist (pre-launch)?

6. **[Modo B Apenas] Design Importado**  
   - URL ou arquivo do design Stitch exportado
   - Formato: PNG, SVG, ou HTML

### Seleção de Modo

- **Modo A:** Pure Claude (todos agentes criam do zero — comparação)
- **Modo B:** Hybrid (padrão — design importado + código Claude)

## Checklist do Usuário

- [ ] Preenchi todas as informações acima
- [ ] [Modo B] Arquivo de design foi fornecido
- [ ] Entendo qual modo estou usando (A ou B)
- [ ] Estou pronto para aprovar o briefing no step-03

## Output

Espera pela aprovação para avançar ao Step 02 (Beatriz Briefing).
