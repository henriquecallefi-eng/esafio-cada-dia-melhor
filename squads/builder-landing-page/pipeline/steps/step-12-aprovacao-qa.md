# Step 12 — Aprovação QA

**Type:** Checkpoint  
**Agent:** User  
**Input:** `auditoria.md` (Rodrigo)  
**Output:** Approval ou fix requests

## Descrição

User revisa a auditoria de Rodrigo. Se todos os scores são >= 7/10, aprova. Senão, solicita fixes.

## Checklist

- [ ] Conversão >= 7/10? (CTA funciona?)
- [ ] Acessibilidade >= 7/10? (Contraste OK?)
- [ ] SEO >= 7/10? (Title, meta presentes?)
- [ ] Performance >= 7/10? (Lighthouse OK?)

## Se Todos >= 7/10

Clique "Aprovar para Deploy" → Step 13 (Pedro Deploy).

## If Algum < 7/10

Descreva qual eixo precisa fix. Ana ou Henrique executarão novamente conforme necessário.
